start:
    EBDA를 뺀 메모리 끝부분(640k-EBDA-second)에 자기복제후 점프

continue:
    시리얼포트를 초기화하고 LI 출력 (first에서 한것) 시리얼 포트가 활성화됐으면 이제부터 serial port쪽에도 출력한다.

drkbd:
    키보드 버퍼를 비운다.

comcom:
    LILO의 세번째 글자 L을 출력 
    디스크 파라미터의 섹터수가 9이하라면 18로 patch 한다. (720K -> 1.44M?)

dskok:
    timeout 핸들러설치

restrt:
	세그먼트 세팅 및 계산
    second와 0x9000 사이에 32kb 공간이 있으면 initseg=0x9000 아니면 initseg=cs-32kb

restrt1:
    스택을 initseg 아래로 세팅한다.
    문자열(LILO, MAGE), 스테이지(2),버전(23.2) 비교
	부팅된 하드와 저장된 하드가 다르면 devmap에 추가한다.

ldsc:
	first와 second의 map 생성시간 비교
	키보드 테이블을 읽는다. 여기엔 각종 정보가 담긴 Menutable을 포함된다.
	
	매핑용 볼륨 테이블을 만든다. 결과는 devmap,vtab,rtab등에 들어간다.
	devmap는 logical:physical 형태로 이루어진다.
	vtab, rtab에는 연결된 하드의 raid_offset이 들어가고 rmask는 해당 하드가 레이드인지 비트로 표시한다.

	디스크립터 섹터수만큼 로드하고 crc32 체크한다.	

	vmware가 동작중인지 체크한다.

vir_loop:
	vmware면 VMDISABLE가 켜진 디스크립터를 지운다. (뒤쪽 디스크립터들을 한칸씩 당겨쓴다.)

virtual_done:
	call kbtest ; 키보드가 연결되었는지 확인한다.
	키보드가 연결되지 않았으면 NOKBDISABLE이 켜진 디스크립터 삭제

kbd_done:
	first의 플래그와 menutable의 플래그 OR
	keytable에 저장된 섹터주소로 default command line를 읽어온다.
	DFL의 매직넘버가 DC_MAGIC(0xf4f2)면 "mk"를 덮어쓰고 DFL 섹터를 디스크에 쓴다. DC_MAGIC이 아니면 cmd line(DFL+2)에 null을 넣어서 command line을 무효화한다.
	에러가 나면 ldsc부터 다시 시작
list:
dokay:
	"O 23.2 "를 한번만 출력한다.
	[hma]=0 ; old_delay에 delay를 or해주고 delay=0
	기본으로 분기할 곳(nodfl)에 iloop를 넣어둔다.
kbd_present:
	prompt 옵션이 켜져있는지 체크한다.
skip_prompt:
	키보드와 시리얼이 없거나 prompt 옵션이 꺼져있다면 마지막으로 키가 눌렸는지 체크한다(시리얼포함). lilo는 prompt 옵션을 꺼도 shift,alt,ctrl등 키를 누르면(시리얼은 break signal) prompt로 들어간다.
	키가이 안눌렸다면 다음 분기점은 키입력 없이 bfirst가 된다.
extp:
	extp(external parameter)를 체크한다. 값이 있다면 niloop로 간다.
noex:
	extp을 사용하지 않을때의 값들을 세팅해준다.
	es=cs
	si의 Dflcmd+2가 null이 아니면 niloop로 간다.
	아니면 미리 넣어둔 분기점(iloop,bfirst)으로 점프한다.
iloop:
	메모리 옮기기 실패시 에러메세지 출력후 restrt로 간다.	
	message=옵션으로 메세지가 정의되었으면 디스크에서 메세지를 읽어온후 화면에 출력한다.
	si= UI_MAGIC(0xff)
niloop:
	"boot: " 문자열 출력
	bx=cmdline
clend:
	cmdline(코드부분 데이터끝)의 값을 null이 올때까지 출력한다.
	prechr(BOOT_IMAGE다음)에 공백 " "을 넣는다.
input:
	실제 키입력부분. [si]가 UI_MAGIC(0xff)면 키 입력을 받아 처리하고 그 외 값이면 저장된 옵션을 처리한다.
kbinp:
	키입력을 받는다. timeout안에 키입력이 없으면 brto로 점프
	키입력의 null은 무시한다. 저장된 cmdline에서는 null은 끝을 의미한다.
gotinp:
	특수키들을 먼저 처리한다. (null,bs,^u,tab등등)
